## R1：一天一篇 + 強制輕重交替（v0.7.2 細化）

- **上限** 24h 內 Mode B 最多 1 篇（多發 = 觸及降 30-50% / 演算法判 spam）
- **下限** 不跳過任何一天（跳過 = 活躍訊號 decay）
- **Mode B 純血 = 嚴格一天一篇**；**Mode C 深度反思 = 同日 2 篇可行**（見 R29）
- **工業 baseline 3-5 篇/週**，Hao 7 篇/週 = 2x 過頻 risk → 每週 ≥ 2 篇非 AI 主題（作品/旅遊/日常）
- 連 3 🔴 同敘事意圖 = 演算法降權；換新意圖則 OK（見 R5）

## R2：爆款後 24h 冷卻

爆款（讚 ≥ 100 或讚 ≥ 50 且留言/讚 > 0.5），24h 內禁發 F6b / F3 長文。該做：回留言、拉群、單則作者精選留言置 CTA（> 50 留言時）。不自動大量回留言（> 30 則重複觸發 spam）。

## R3：突破鐵粉圈（連 3 篇 < 15% 非追蹤者）

演算法歸類「鐵粉限定」。下一篇必用 F8-F13 廣推或 Mode C（F22/F23 實證可 broke 90%+）。F8 tag 外部 / F12 時間戳 / F10 居中開炮（週 ≤ 1）/ F11 反漏斗（月 ≤ 1）。

## R4：時段按受眾分流

AI/tech 創作者受眾夜貓族，FB 黃金 22:00-01:00（實證 02:13=72K / 09:48=373）。Mode A 不挑時段。Thread 不挑時段。

## R5：敘事意圖冷卻（核心規則，5 案例升級）

**「主題」= 敘事意圖**（promo / 演算法復盤 / 商業 ROI / 社群 social proof / 教學 / 合作 / 預告 / 抱怨...），不是公式 / hook 詞。

- 同敘事意圖 **4 天內不重複**（違反 = self-promo 疲勞）
- 同敘事意圖 **每月 ≤ 2 次**
- 不同敘事意圖**完全不限頻率**（5/5 月內第 5 個 F6b 但新意圖 → mega-viral 44K）
- **月配額單位 = 敘事意圖，不是公式總數**（v0.6 推翻 v0.5 R9）

| 貼文 | 意圖 | 月內 | 距前次 | 結果 |

## R6：評估看 4 指標 + 必須 48-72h plateau 才判定

- **1-6h**：不下結論（underestimate 60-80%）
- **6-24h**：只說趨勢
- **48-72h plateau**：才用 4 指標判定

**4 指標**：連結點擊率 > 1% / 追蹤者比例 / 互動率 > 15% / 下游轉化（入群/star/付費）。

**多次實證早判失誤**：Day 2 / Day 4 / Day 5 / 5/12 F19 都後期翻盤。**1-19h 絕不下定論**。

## R27：個人脆弱 confess（Mode C 廣 identity）

揭露真實卡點/焦慮/拖延（不是炫成就）= broke 鐵粉圈 90%+。「我一直想 X，最大卡點是 Y...拖了大半年」= vulnerable + relatable + 普世議題。F22 實證 91.5% 非追蹤。

## R32：集體 framing trigger（v0.9.3，5/26-5/29 三實證）

**任何 Mode B / Mode C 篇用「我們 / 大家一起 / 不只是我」反個人主義 framing = broke 鐵粉圈廣度 +30-50%**。

實證：
- F25a 5/26 ship 系列承諾：**39.7% 非追蹤 broke**（12 儲存）
- F25b 5/28 Line 5000「不是我一個人的功勞」：留言 quality 高
- F25c 5/29「一個人快，一群人遠」階段 review：20.4% 非追蹤

vs 個人 hype：F6b「我殺瘋了」mega-viral 但稀有；F25 集體 framing **普遍可達 broke 鐵粉圈**。

**何時用**：
- 想 broke 鐵粉圈但缺 mega hook 時
- 鐵粉黏著疲勞 → 用「我們」感重燃歸屬感
- ship 系列 / 達 milestone / 階段 review 等敘事時刻

**避免**：
- 不要假大方（要真實 + 具體歸功對象）
- 不要連發 F25 系列（洗鐵粉）
- 不要在 R31 Brand 邊界澄清後立刻用（避免「我們很好我們很棒」連續式自誇）

## R31：Brand 邊界澄清時機（v0.9.2，5/27 F24 實證）

當以下任一觸發 → 用 F24 邊界澄清型：

1. 私訊量爆（回不完）
2. 陌生交友邀請太多
3. 有人誤解你在賣課 / 接案 / 帶貨
4. 鐵粉開始有「跟我們有距離」感
5. 月內 hype + ship 內容過多後需要 reset 信任

**頻率**：每月 1 次最多（太頻繁 = 自我中心感）。

**voice 鐵則**（跟其他 Mode C 不同）：
- 無「！」hype
- 無 TM 級用詞
- 無攻擊性動詞
- 用「── 真的沒有」溫和強調（「──」破折號緩衝）
- 每點 numbered（一、二、三、四、五）
- 謙卑收尾「感謝配合 + 謝謝支持」

**實證 5/27 F24**：2,754 觀眾 / 42.4% 非追蹤 / 173 讚 / **84 留言** / $0.19 收益。鐵粉強認同（「太猛了大神」「感恩分享」）。

## R30：FB 社團 cross-post 策略（v0.9.1，5/22 + 5/29 實證）

主帳號 timeline 發完後，cross-post 到 AI / dev 主題社團 = **留言放大 5-10x / 觸及 2-3x**。

**操作鐵則**：
1. 主帳號自己 timeline 先發（讓鐵粉看到原始版）
2. **1-2 小時後** cross-post 到主題對口社團
3. 同篇文不重寫（社團 audience 不重疊）
4. **每篇選 1-2 個社團**，不洗 3+ 社團（spam 警告）
5. 主動回留言區 = 社團 algorithm 加權

**社團對口 audience**：

## 🎯 Viral 4 條件公式（核心，5 案例 AND 實證）

```
viral = 4 段 4 句結構 + 純血 voice + 全新敘事意圖（R5）+ 黃金時段（R4）
```

4 個 AND 不是 OR，任一缺 = 死。但 **Readability 是隱藏第 5 條件**（5/12 V3 四條全綠仍 fail，因 meta 4 層看不懂）。

## 📝 Readability 三項硬檢查

1. **5 秒非追蹤者讀懂測試**（meta 層數 ≤ 1 最佳，≤ 2 上限）
2. **meta 層數 ≤ 2**（第 3 層以上鐵粉懂新觀眾爬不過）
3. **數字密度 ≤ 3 個/段**（超過 = dashboard report）

## 📦 FB compose 段落硬規則

4 段 4 句貼 FB 必須手動兩次 Enter 保段落（FB 預設壓掉單 Enter）。給草稿時必標註「段間按兩次 Enter」或用 --- 替代。
