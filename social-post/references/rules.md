## 📐 證據強度公約（升級門檻，不佔 R 編號）

全 repo 通用的升級標準。任何規則、公式、案例結論要標成 proven，都要過這一關。

| 標記 | 門檻 | 寫法限制 |
|---|---|---|
| **proven** | **n ≥ 2 個彼此獨立、confound 可分離**的案例 | 可寫成鐵則、可加粗 |
| 🧪 **emerging** | n = 1，或 n ≥ 2 但 confound 糾纏無法分離 | **不准寫成鐵則、不准加粗**，要標出未分離的變數 |
| housekeeping | 純一致性修正（數字對齊、錯字、交叉引用） | 不涉及強度宣稱 |

**獨立的定義**：不同帳號、不同題材、不同話題波、不同時段 —— 共用其中任何一項就要說明，共用多項就不算獨立。

**兩條鐵律**：
1. **沒有出處的數字一律刪**，不要因為「看起來合理」就留著。內插出來的數字（在兩個已知點之間補中間值）**不算實證**。
2. **沒有新增也是合格的結論。** 審計的價值在於找出該刪的，不是產出更多規則。

## R1：一天一篇 + 強制輕重交替（v0.7.2 細化）

- **上限** 24h 內 Mode B 最多 1 篇（多發 = 觸及降 30-50% / 演算法判 spam）
- **下限** 不跳過任何一天（跳過 = 活躍訊號 decay）
- **Mode B 純血 = 嚴格一天一篇**；**Mode C 深度反思 = 同日 2 篇可行**（見 R29）
- **工業 baseline 3-5 篇/週**，Hao 7 篇/週 = 2x 過頻 risk → 每週 ≥ 2 篇非 AI 主題（作品/旅遊/日常）
- 連 3 🔴 同敘事意圖 = 演算法降權；換新意圖則 OK（見 R5）

## R2：爆款後 24h 冷卻

爆款（讚 ≥ 100 或讚 ≥ 50 且留言/讚 > 0.5），24h 內禁發 F6b / F3 長文。該做：回留言、拉群、單則作者精選留言置 CTA（> 50 留言時）。不自動大量回留言（> 30 則重複觸發 spam）。

### R2 擴充：冷卻要看「爆款有沒有停止成長」，不是死抓 24 小時（🧪 emerging）

爆款仍在爬坡時發同敘事意圖的下一篇，可能被吸走演算法配額。實證觀察：7/20 value-prop 長文到 33h 仍在漲（非追蹤 81% → 83.8%），7/21 約 24.5h 後發同屬反賣課主題的立場論述文 → 9h 僅 31.7% 非追蹤（38h 回升至 45.5%，但瀏覽 7,214 仍只有 mega 的一小部分）／ 分享 10（Case 43）。⚠️ 三因糾纏（沒 artifact ／ 配額被吸 ／ 同題材疲勞）無法分離，標 emerging。**操作上：前一篇還在漲就多等，或至少換敘事意圖（R5）。**

## R3：突破鐵粉圈（連 3 篇 < 15% 非追蹤者）

演算法歸類「鐵粉限定」。下一篇必用 F8-F13 廣推或 Mode C（F22/F23 實證可 broke 90%+）。F8 tag 外部 / F12 時間戳 / F10 居中開炮（週 ≤ 1）/ F11 反漏斗（月 ≤ 1）。

## R4：時段按受眾分流

AI/tech 創作者受眾夜貓族，FB 黃金 22:00-01:00（實證 02:13=80K / 09:48=373）。Mode A 不挑時段。Thread 不挑時段。

### R4 擴充：時段 buff 不可疊加重複 + 內容夠強可破時段

**時段是必要不充分（兩條 proven 警告）**：
1. **時段 buff 不可重複疊加**：Day1 mega buff = 「新鮮主題 + 凌晨 + 演算法首推」三疊加；Day5 同 02:13 只剩凌晨一項 → flop 0.13%（Case 5）。**凌晨時段本身不帶 viral，要配新意圖（R5）**。
2. **內容夠強可破時段弱點**：5/12 F19 下午 2-3pm（非夜貓時段）仍 broke 814（Case 10）= 內容 > 時段。**7/20 星期一中午 value-prop 長文打到 83.4% 非追蹤 ／ 分享 193 ／ 儲存 303（Case 42，62h 準 plateau），是目前最硬的一次「白天照樣 mega broke」實證**；同批 7/16 星期四中午圖卡也有 51.1%。**時段是加分項不是門票。**

**操作**：Mode B 爆款型排 22:00-01:00；白天有 ship moment 用 F19 mini（內容強可破時段）。Mode A 日常 / Thread 不挑時段。

## R5：敘事意圖冷卻（核心規則，5 案例升級）

**「主題」= 敘事意圖**（promo / 演算法復盤 / 商業 ROI / 社群 social proof / 教學 / 合作 / 預告 / 抱怨⋯），不是公式 / hook 詞。

- 同敘事意圖 **4 天內不重複**（違反 = self-promo 疲勞）
- 同敘事意圖 **每月 ≤ 2 次**
- 不同敘事意圖**完全不限頻率**（5/5 月內第 5 個 F6b 但新意圖 → mega-viral 44K）
- **月配額單位 = 敘事意圖，不是公式總數**（v0.6 推翻 v0.5 R9）

| 貼文 | 意圖 | 月內 | 距前次 | 結果 |
|---|---|---|---|---|
| Day 1 | promo | 1/2 | — | mega 80K |
| Day 5 | promo | 2/2 | 4 天 | flop 0.13% |
| Day 6 | 演算法復盤 | 1/2 | — | mega 17K |
| 5/5 | 社群 social proof | 1/2 | — | mega 44K |

⚠️ **題材系列疲勞**（v0.8.x 觀察）：連 5 篇繞「Line 群數字」鐵粉疲勞 → 同題材系列月內 ≤ 3-4 次。

**🏆 6/9-6/15 giveaway 系列實證（demo > claim 可救疲勞）**：
- 6/9 全公開（42K/93.1%）→ 6/10 POS（1,510/21.2%）→ 6/11 上午網紅製造機（2,153/**15.1% 鎖鐵粉**）= giveaway intent 連發第 3 次疲勞（R35「我要」衝 91 留言仍不 broke）
- **但同日 6/11 下午自動剪輯 demo（7,584/68% broke / 45 儲存）** = 同 giveaway 主題，換成**具體 demo + 看得到的成品 + Fable 5 news hook** 立刻 broke
- 6/13 Fable 5 被關（news+vulnerable，無 CTA）48.9% broke = 換 intent 也救
- **結論**：intent 疲勞時靠（1）換 intent 或（2）**具體 demo + news hook** 重新 broke。**抽象 claim（「我很強」）= 鎖鐵粉；具體 demo（「看成品」）= broke**。comments ≠ broke（91 留言抽象篇仍 15.1%）。

### R5 補充：兩條疲勞軸（非矛盾）+ 題材軸 superlative 救援 + override 風險分級

**⚠️ 兩個獨立配額軸（非矛盾）**：「月 ≤ 2」= intent 月配額；「giveaway 連發第 3 次必鎖鐵粉」= 連續發文閾值（6/9→6/10→6/11 實證 93.1%→21.2%→15.1%）。兩者並行，取較嚴者。

**題材軸疲勞（獨立於 intent 軸）**：同題材（Line 群數字 / Discord milestone）連發太多次，即使 intent 各異鐵粉仍疲勞 → 同題材系列月內 ≤ 3-4 次。第 4 次同題材必加 **superlative（R41）/ 集體 framing（R32）/ 第三方戰績（R38）** 重新 broke（5/28 純數字 16.7% → 6/14「台灣最大」57.9%，3.5x）。

**override 風險分級**（meta 規則）：R5 主題類 override = **高風險**（Day 5 override 冷卻 → 0.13% flop）；voice consistency override = **低風險**（Day 6 鎖純血 → broke 57%）。override 前按此分級提示風險，仍尊重執行 + 標記戰績備註 + 降風險混搭。

## R6：評估看 4 指標 + 必須 48-72h plateau 才判定

- **1-6h**：不下結論（underestimate 60-80%）
- **6-24h**：只說趨勢
- **48-72h plateau**：才用 4 指標判定

**4 指標**：連結點擊率 > 1% / 追蹤者比例 / 互動率 > 15% / 下游轉化（入群/star/付費）。

**多次實證早判失誤**：Day 2 / Day 4 / Day 5 / 5/12 F19 都後期翻盤。**1-19h 絕不下定論**。

**⚠️ 互動率 > 15% 只適用深化型，別拿去砍擴散型**：擴散型（高觸及低互動率）跟深化型（低觸及高互動率）是兩種成功，不可互比。同口徑（讚 + 留言 + 分享 + 儲存 ÷ 瀏覽者）下，Day 1 mega 也只有約 1.3%，7/16 圖卡 news 是 2.79%。**看到擴散型的低互動率別急著判失敗，先看非追蹤比例。** 反過來也成立：**深化型的低非追蹤比例也不是失敗**（互動率高 + 讚美型留言 = 鐵粉做深）。⚠️ 但 Case 43 是**兩把尺都會誤判的教材**：9h 時 31.7% ／ 互動率 9.54% 看起來是純深化型，38h 比例爬回 **45.5%** —— 早判連「屬於哪一型」都會判錯，先等 plateau 再選尺。 另外算互動率時分母要統一用「瀏覽者」，混用瀏覽次數會得出假的高低。

### R6 擴充：second push 機制 + niche/普世 reach 分流

**① 演算法 second push（24-48h 第二波廣推）**：第一波只推鐵粉測水溫；24-48h 演算法看互動信號（留言/儲存累積 + 主題夠新鮮 + 值得分享）決定要不要第二波廣推給非追蹤者 → 非追蹤比例從個位數跳到 29-30% broke。Day4：1h 6% → 58h 29% broke；5/12 F19：6h 13.8% → 24h 29.8% broke（2.73x）。**沒觸發 = 主題重複 / 判 self-promo**。

**⚠️ 非追蹤比例會「往上爬」= 第二波還在放量，別急著收單**：一般長尾是追蹤者先看完、陌生人零星補 → 非追蹤比例**往下掉**；若比例**往上爬**代表第二波廣推正在放量，還沒到頂。實證 n=2：7/20 value-prop 長文 12h 81% → 33h **83.8%**（Case 42-C）；7/21 立場論述文 9h 31.7% → 38h **45.5%**（Case 43，且分享儲存幾乎沒動，漲的全是演算法自推）。⚠️ 兩例同帳號同週，獨立性打折。**看到比例上升就再等，這時候記的數字一定會被自己推翻。** 🧪 收尾訊號（n=1）：比例**轉平或微降**（Case 42-C 33h 83.8% → 62h 83.4%）= 第二波放量結束、plateau 到了，可以收數字。**比例的形狀就是廣推生命週期圖。**

**② niche vs 普世 reach 分流（評估前先判天花板）**：niche 單品只對相關 segment 有用 → 觸及**天生封頂**（POS 1,510 vs 普世 giveaway 42K = 28x）= 先天封頂不是打法失敗。niche → 看 segment 內 save/share 品質（POS 各 1.1% = 對的人）+ 下游轉化（R7），不看絕對觸及；加 R35 keyword CTA 可 1,510→3K-8K。

**兩種成功不可互比**：擴散型（高觸及低 conversion）vs 深化型（低觸及但連結點擊率 2.3% 隱藏導流王）── 深化型「讚暴跌」≠ flop。

## R7：真 KPI 是社群轉化，不是讚

Day 1：448 讚是表面，**Line 群 800→2,119（+1,319）**才是真 ROI。看 downstream：社群人數 / star / 付費學員。社群成長含多平台貢獻，歸因不要全歸單篇。**Line 群整月 800 → 4,568+（127/day avg）= 真 KPI 加速度持續**。

### R7 擴充：Community 飛輪閉環 + 數字回落 ≠ 失敗

**① 飛輪閉環（milestone 自己滾雪球）**：FB/Thread viral → R35 keyword CTA（留言「Discord」）→ 大量留言（5x 權重再 broke）→ 公開 link 自助 → 私域灌入 → 衝新 milestone → 發 milestone 文（F25b + R41 superlative）→ 再 broke + 再導流 → 回頂端。實證 6/9-6/14：6/9 giveaway（42K/959「Discord」留言）→ Discord 6,930→8,887（台灣最大）→ 6/14「台灣最大」57.9% broke。

**② 社群數字回落 ≠ 失敗**：私域非單調成長，回落多為 quality 自篩。
- **假帳號清洗**：Line 過 5,000 後假帳號自然退（5,030→4,998）= 社群文化自我篩選，別誤判成內容失敗
- **平台留存脆弱度**：Line（達標自然清洗）/ FB 鐵粉（脆弱，voice 套路化會 unfollow，連 R42）/ Discord（最黏）/ Thread 追蹤（viral bonus 過後流失）
→ 看回落先判是不是清洗（過 milestone 後多半是），不對清洗做補救式發文。

**真 KPI 公式**：`入群人數 × 活躍佔比 × 留存率 × 轉化次留`，非絕對人數。

### R16 補充：留言量 ≠ broke（CTA 是必要不充分）+ 開放問句磁鐵 + 選詞

**🚨 留言量 ≠ broke**（同 R36/R39）：keyword CTA（R35）能把留言堆高（鐵粉最會留言），但留言量**不等於 broke**。演算法 broke 看「內容值不值得給路人」（具體 demo + 儲存 R18 + 分享 R39），不是留言數。
- 血淚（Case 33 同日 A/B）：A 篇 91 留言（4x D 篇）broke 卻只 D 的 1/4（15.1% vs 68%）。
- **鐵則**：下 R35 keyword CTA **前**先過一關 ── 內容是「具體看得到/可驗證的東西」嗎？是 → R35 放大；只是「相信我很強」抽象 claim → 先改成具體 demo 再下 CTA。

**拉長留言 hook**：開放式提問 / 觀點挑戰 / 經驗請求 / **🏆 開放問句磁鐵**（「正在考慮要不要免費送，有沒有興趣」F26，6/8 衝 51 留言 5% rate）。社團 cross-post 留言 5-10x（R30）。

**keyword 選詞**：短（1-3 字：自動化/我要/Discord/POS/+1）/ 好打不切輸入法（避英文長詞）/ 語意=終點或索取物（「Discord」=終點衝 959；「POS」=單品名自篩 niche）。

## R34：真實 voice — 反 AI 腔（🚨 硬規則，6/1 使用者鎖定，永久不可覆寫）

**起因**：有讀者說 Hao 最近的文「很像 AI 文」。

- 真正像 AI 的是 **抽象空詞 + staged 套路 + over-narrate + 過度工整**

### A. 標點：日常 voice 以 ，、為主，proven 公式裝置照用

- **預設**（日常 Mode A / Mode C 反思 prose）：用 `，`（逗號流）`、`（頓號），句子靠換行斷，不用 `。 ？ ： ； …`
- **proven 公式裝置例外**（真實爆款驗證過，照其公式規則用，不是 AI 腔）：
  - F6b / F6a / F15 mini hype：`！！！`（Day1 / 5-5 mega-viral）
  - Mode C F21 / F23：`── 段落標題 ──` 分隔符（R26，5/22 broke 94.5%）
  - F22 / F25：`──` 做轉折 + `！！`（5/22 broke 91.5%）
  - F19 Thread：`！` / `！！`（25/7 26 萬瀏覽，R19）
- **永遠不用**：`：` 戲劇冒號做開場 setup、英文標點、半形

### 全形鐵則（半形只留給 code 跟數字）

- 中文語境內所有標點一律全形：，、。！？：；（）「」
- 半形只允許出現在四種地方：
  1. 程式碼（三反引號圍的區塊與行內 `code`）
  2. URL、檔名、路徑（references/rules.md 這種）
  3. 英文詞與英文句子內部的標點
  4. 數字千分位與小數點（42,103 / 17.4 這種絕對不能動）
- 其他地方看到半形標點，直接改全形，不用猶豫

### B. 反 AI 腔黑名單 ⭐ 真正的元兇（所有 voice 都適用，出現即重寫）

- **抽象空詞**：護城河、本質、複利、賦能、底層邏輯、真正的 X、核心競爭力、長期主義、未來已來、降維、沉澱、佈局、賦予意義、儀式感、價值升格
- **staged 開場**：「說個我以前看不懂的事」「分享一個觀察」「講個故事」「你有沒有發現」
- **過度對仗**：「不是 X，而是 Y」連用兩次以上、工整排比三連
- **over-narrate 情緒**：「那種感覺真的會讓你⋯」「那一刻我突然⋯」
- **空泛升格收尾**：「永遠值錢」「未來已來」式格言 ── 除非像 Hao 真實的「**會設計製作流程的人**永遠值錢」那樣**綁具體技能**，否則砍

- **口語詞**：弄、搞、幹嘛、想通、好像、有夠、扯、草、爽

- **不對齊 = 人味**：真人深夜不會寫對仗句，會講一半跳下一句

### R34 補強 1：mental model（避免「要真實就不能 hype」誤解）

**🧭「語氣強度」與「反 AI 腔」是兩個正交軸** ── 可以**又 hype 又不像 AI**。proven 公式裝置（！！！/ ── / TM 級用詞）是 organic creator signal **不是** AI 腔（Case 30：F6b hype ！！！+ R34 乾淨詞 = mega 42K/93.1%/959 留言）；真正 AI 腔是**抽象空詞 + 過度工整**。別把「要真實」誤解成「不能 hype」。

### R34 補強 2：不對齊硬檢（過度工整 = 最隱形 AI 腔）

**機制**：對仗排比讀起來順、有 punch，所以最容易漏檢，但真人深夜寫字會「講一半跳下一句」，不會三連對仗。過度工整 = 演算法 + 老讀者都聞得到的 AI 腔。

**硬檢三問（任一中 = 重寫該句）**：
1. **對仗連用**：「不是 X，而是 Y」「X 是…，Y 是…」出現 ≥ 2 次 → 拆掉一組，改不對齊口語跳接
2. **排比三連**：三句同結構並列（「能…的人、會…的人、懂…的人」）→ 留 1 句，其餘改散句
3. **空泛升格收尾**：「永遠值錢 / 未來已來 / 真正的 X」格言式結尾 → 除非綁**具體技能/動作**（Hao 真實「會設計製作流程的人永遠值錢」），否則砍

proven 公式裝置（F6b ！！！/ Mode C ──）不算工整違規（那是節奏不是對仗）。

### R34 補強 3：口語密度配額 + 自嘲 > 宣言

**口語詞配額（Mode A / Mode C prose）**：每篇至少 **2-3 個口語動詞 / 語氣詞**（弄、搞、跑、改、debug、ship、想通、好像、有夠、扯、草、爽、幹嘛）。抽象動詞替換：封裝→弄成/塞進、賦能→幫你做、沉澱→留下來、優化→調/改。

**自嘲 > 宣言（觸發場景）**：
| 場景 | 宣言版（弱）| 自嘲版（broke）|
|---|---|---|
| 工具更新 | 「我又更新了」| 「Claude 幫我留一手，閹割版被我抓到」（Case 36, 54.2%）|
| 卡點/失敗 | 「我克服了」| 「拖了大半年沒動」（F22, 91.5%）|

## R36：demo > claim 鐵律（broke 鐵粉圈最強單一槓桿，🏆 4 案例同變數實證，跨 FB/Thread）

**這是整個 broke 系統的母規律**，散在 R5 giveaway 段 + Case 33/37/38/39 的單一最高頻 broke 根因，提取為獨立硬規則。

**觸發時機**：任何 giveaway / promo / social proof / ship / skill 推廣篇。寫完草稿先自問一句 ──「我是在給路人**看得到、能驗證**的成品，還是要他**相信我很強**？」後者 = 鎖鐵粉。

**機制**：演算法 broke 判定看「內容對路人值不值得」= **可驗證成品 + 儲存（R18）+ 分享（R39）**，**不看留言數**。
1. 抽象 claim（「我很強」「很多人用我變網紅」「我 skill 讓人變網紅」）= 路人無評估依據 → 只有鐵粉買帳 → **鎖鐵粉**
2. 具體 demo（「看，這支是 Claude 剪的」+ 留言區成品 / 一個誇張可查數字）= 路人能自己判斷有沒有用 → 演算法廣推 → **broke**
3. **🚨 comments ≠ broke**（核心反直覺）：抽象篇靠 R35「我要」可衝爆留言（91）但仍 15.1% 鎖鐵粉；具體 demo 留言少（24）broke 卻 68%。留言量是 broke 的**助燃劑不是點火器**。

**鐵證 A/B（同變數對照表）**：
| 對照 | 抽象 claim | 具體 demo |
|---|---|---|
| 6/11 同日同帳號 A/B（Case 33）| 網紅製造機 2,153 / **15.1% 鎖鐵粉** / 91 留言 / 20 存 | 自動剪輯 demo 7,584 / **68% broke** / 24 留言 / 45 存 / Reel 1/10 |
| 6/26 social proof（Case 37）| 「很多人用我變網紅」15.1% | 「夥伴一篇衝 17.4 萬 / 400→4,600」**83.3% broke** / 53 分享 |
| 6/25 開頭（Case 38）| meta 故事 4,549 / 54.2% / 22 存 | value prop「丟素材→自動剪+雙語字幕」**38,188 / 90.4%** / 466 存 |
| 6/25 Thread（Case 39）| 抽象「我免費分享技術」26 萬 / 轉發 76 | 綁「100 剪輯記憶 + 自動剪輯」**plateau 49.5 萬 / 轉發 1,278（16.8x）** |

**hook 三層級（弱→強）**：claim「我很強」< curiosity「好奇 hook / meta 故事」< **demo / value-prop「看成品 / 它幫你做什麼」**

**具體操作**：
1. 永遠 **lead with value prop**（它幫你做什麼）> meta 故事 > 抽象自誇（細部開場策略見 R37）
2. 把成品 / 可驗證 demo 放**留言區**（R25 合規：正文無 URL）讓路人能驗證
3. social proof 用**第三方具體戰績**（R38）> 自我抽象宣稱；綁「一個誇張可查數字」（17.4 萬 / 400→4,600）不講「很多人用我」
4. Thread F19 stance 綁「當下超火的具體 artifact」（100 剪輯記憶）而非抽象立場
5. 抽象自誇詞（暴漲 / 很強 / 網紅製造機）= 紅旗，改成可驗證數字 / 看得到的東西
6. 可疊 **news hook**（「Fable 5 一出我馬上燒 60 鎂優化」）= 好奇 + 具體投入誠意

**配額 / 限制**：無配額（是每一篇動筆前都要過的 framing 檢查，不是公式），但受 R5 敘事意圖冷卻管轄。

**🏆 救援用法**：R5 intent 疲勞鎖鐵粉時，**換成具體 demo 可立刻重新 broke，不必換 intent**（6/11 同日同主題實證：上午抽象 15.1% → 下午具體 demo 68%）。與 R37 value-prop-lead 互為表裡（value-prop = 講清楚它幫你做什麼；demo = 直接給你看成品）。

### R36 補充：「可驗證」不一定要有成品 —— 可以外包給讀者自己的後台

R36 的核心是「路人能不能自己判斷這是真的」，**不必然要有 demo 產物**。7/16「Meta 幾乎所有用戶 30 天流量都公開顯示了」= 每個讀者打開自己的洞察報告，三秒內就能自證，驗證成本近乎零、而且不需要相信作者 → 沒有成品、沒有儲存、沒有公開分享，仍 broke 51.1%（Case 42-A）。對照 Case 33-A「相信我很強」的抽象 claim 只有 15.1% 鎖鐵粉。**平台事實 / 人人可查的改動 = demo > claim 的無成品變體。**

## R37：value-prop-lead 開場 —「它幫你做什麼」碾壓 meta 故事 / 好奇 hook（🏆 6/25 實證，史上最高儲存）

**R36 母規律在「開場句」層的具體展現**：不是「成品 vs 宣稱」，是「開頭講用途 vs 開頭講故事」。

**鐵律**：giveaway / 工具 ship / 開源物推廣篇，**開頭第一段直接講「它幫你解決什麼具體的事」+ 用法**，不要用 meta 故事（「Claude 幫我留一手」）或好奇 hook 開場。

**觸發時機**：
- 任何「送一個有用工具 / 開源 skill / ship 一個產品」且工具有**明確 value prop**（幫使用者做完一件具體有用的事）
- 受眾普世（每個做這件事的人都想要，如自動剪片 = 每個做影片的人）時效果最大
- 跟 R35 keyword CTA、R36 demo>claim 一起用

**機制**：
1. 清楚 value prop = 每個目標受眾**立刻判定「這對我有用」**→ 演算法判「內容值得給路人」→ broke
2. 「具體有用 + 怎麼用」→ 大量**儲存**（先存著之後用，R18 次強信號拉滿）
3. meta 故事要**先讀完才知道有沒有用** = 摩擦；value prop = 0 摩擦判定
4. curiosity hook 有天花板：純好奇 < value-prop（決策樹：普世 ship 篇優先 value-prop；產品自證 / 復盤 / 私藏 reveal 才用 curiosity meta hook）

**實證 A/B（同一個自動剪輯工具，唯一變數 = 開頭）**：
| | 6/23 meta 故事開頭（Case 36）| 6/25 value-prop 開頭（Case 38）|
|---|---|---|
| 開頭 | 「Claude 之前幫我留一手」meta 故事 | 「丟素材→自動分析+雙語字幕+對齊+自動剪，不用動手」value prop |
| 觀眾 | 4,549 | **38,188（8x）** |
| 非追蹤 | 54.2% | **90.4%** |
| 儲存 | 22 | **466（21x，史上最高）** |
| 分享 | 8 | **195（史上最高）** |

**開頭模板**：
```
[第 1 句] 我的＜工具＞全部開源了
[第 2 句 value prop] ＜輸入＞丟進去，自動＜動作1＞、自動＜動作2＞、＜對齊/品質保證＞，整支幾乎不用動手
[第 3 句 用法] 用法：＜3 步驟最簡流程＞（建專案放素材 → 丟主要素材 → 剩下交給它）
[giveaway + R35 CTA] 全免費，想要的留言『關鍵詞』，連結放留言區
```

**具體操作**：
1. 開頭第一句 = **它幫你做什麼**（用具體口語動詞「丟、剪、對齊」，不是「我發現一件扯的事」、不用抽象空詞）
2. 緊接**清楚用法**（受眾知道怎麼用 = 願意存）
3. value prop 要「每個做這件事的人都想要」的明確度

**何時反而用 meta 故事**：要「換新 intent 避開 giveaway 疲勞」時（如 F29 再ship 揭露角度），meta 故事 broke 54.2% 仍合格；但要 mega（broke 90%+/儲存爆）必用 value-prop 開頭。

**配額 / 限制**：無配額（是開頭寫法不是公式），但受 R5 冷卻管。前提是工具真有清楚 value prop；若是抽象 skill / 理念，value-prop-lead 無物可講，退回 meta / 再ship 角度。niche 單品（只對小眾有用）value prop 再清楚 reach 仍有天花板（POS 21.2%，別用 38K 標準評判，見 R6 niche 分流）。

### R37 補充：純 CTA 公告 = 最差的一種貼文（Case 42 反面教材）

只喊「我要辦 X 了，連結在留言區」而不講「參加能得到什麼」→ 路人沒有點進來的理由、鐵粉沒有轉的理由。實證 7/16 讀書會純 CTA 圖卡：**40.9% 非追蹤（連鐵粉圈都沒破）／ 分享 3 ／ 儲存 2**，是同週三篇裡最差的一篇。**辦活動也要先講價值再給連結。**

## R38：第三方具體戰績 social proof 🧪 emerging（n=1，且與 R36 共用 Case 33-A 對照組） —「別人用我的東西做出的誇張成績」> 自誇（🏆 6/26 實證）

**R36 在 social proof 場景的專用打法**。當你想證明「我的東西有用」── 不要自誇「我很強」，秀**一個具體的、別人用你的東西做出的誇張可驗證成績**。

**觸發時機**：有夥伴 / 學員 / 社群成員用你開源的東西做出可驗證的好成績時。想 broke 鐵粉圈但不想自誇。

**機制**：
1. **concrete > abstract**：具體可驗證數字（17.4 萬 / 400→4,600）比抽象「很多人用我變網紅」可信
2. **works-for-others > I'm-great**：「它對別人也有用」比「我很厲害」shareable、心理阻力低 → 分享自己帶 broke（R39）
3. **謙卑抬高價值**（「磨三個月寫不成 SOP，所以直接開源」）= trust + giveaway 雙觸發
4. comments ≠ broke：抽象篇 91 留言仍 15.1% 鎖鐵粉；具體第三方戰績靠分享自己帶 broke（48 留言但 53 分享 / 83.3%）

**實證對比**：
| | 6/11 抽象自誇（Case 33-A）| 6/26 具體第三方（Case 37）|
|---|---|---|
| social proof | 「很多人用我變網紅」| 「夥伴一篇衝 17.4 萬 / 400→4,600」|
| 非追蹤 | 15.1% 鎖鐵粉 | **83.3% broke（~25K）** |
| 分享 | 15 | **53** 🔥 |

**具體操作**：
1. 秀「一個」他人用你東西做出的成績（一個誇張數字勝過一串）
2. 數字要可驗證、要 striking（17.4 萬 / 400→4,600 比「暴漲」勾人 10 倍）
3. 配謙卑 + 反賣課 giveaway（「我沒辦法教你，直接開源丟給你」）
4. CTA 可降到最低摩擦（「連結放留言區拿去用，不用留言跟我要」R39 反摩擦路徑），內容夠強分享自己帶
5. 公式骨架見 F28

**配額 / 限制**：算 R5「社群 social proof」敘事意圖配額（月 ≤ 2）。**必須真實具體，編造數字 = 信任崩**（R32 假大方禁忌）。第三方戰績難得，有就用。

## R39：分享（Share）= 跨粉絲圈最強引擎（20x 讚 / FB 2026 Unconnected Reach 核心 / 內容強時走分享路徑）

分享權重散在 formulas.md 廣推訊號表，提取為獨立可查硬規則 + 補「反摩擦 CTA」第二條 broke 路徑。

**觸發時機**：任何想 broke 鐵粉圈的 giveaway / demo / 行業洞察 / 社群 milestone 篇。

**機制**：分享權重 **20x 讚**，是「內容值不值得給陌生人」的最強證明 = 觀眾把內容放自己的牆替你背書 = identity 表態。

**🚨 反直覺鐵律：comments ≠ broke**（同 R36）
| 篇 | 留言 | 分享 | 非追蹤 |
|---|---|---|---|
| 6/11 抽象 claim（Case 33）| **91** 🥇 | 少 | **15.1% 鎖鐵粉** |
| 6/11 具體 demo（Case 33）| 24 | — | **68% broke** |
| 6/26 第三方戰績（Case 37）| 48 | **53** | **83.3% broke** |
→ 留言爆 ≠ broke（可能全是鐵粉狂喊「我要」）。演算法 broke 看分享 + 儲存，不是留言數。

**史上分享排名**：6/25 自動剪輯 **195** 🏆 > 7/20 value-prop 長文 **193**（62h，差 2）> Day1 **87** > 6/9 giveaway 37 > 5/5 F6b 35 > F23 25 > F22 20。mega 篇分享 ≥ 35。

**兩條 broke 路徑（R35 keyword-CTA vs 反摩擦 CTA 二選一）**：
| 場景 | 用哪個 |
|---|---|
| 想衝留言量 + 要 Line/Discord 索取 funnel + 內容靠「索取慾」 | **R35 keyword CTA**（留言堆爆 → broke）|
| 內容是具體 demo / 第三方戰績 / 清楚 value prop 超有用工具 | **反摩擦 CTA**（分享自己帶 → broke，留言可低）|

**反摩擦 CTA 操作**：CTA 降到最低摩擦 ──「連結放留言區，拿去用就好，不用留言跟我要，覺得有用順手 GitHub 點顆星」。內容強 → shareable → 分享驅動 broke（6/26 留言僅 48 但分享 53 / 83.3%；6/25 分享 195 / 儲存 466 / 90.4%）。仍要「連結放留言區」（R25），可掛輕量「順手點星」，差別只在不強制「留言關鍵詞」。

**⚠️ 面板的「分享」只計公開分享，私訊分享不進面板**：FB 面板那格是「分享（公開）」，而 **R15 的私訊分享（Messenger）才是最強信號**，它不顯示。所以看到「分享 1」不等於擴散引擎沒點火 —— 特別是**會得罪人的題材**（反講師 / 幸災樂禍），公開轉發等於當眾站隊、掛名成本高，但「傳給那個買過課的朋友看」動機極強 → **最強引擎可能滿載但結構性隱形**。

**⚠️ 第二條 broke 路徑：話題驅動**：借當週熱點（如 Meta 開放流量數據）讓演算法主動推給對該題材有興趣的陌生人，**公開分享只有 1 仍打到 51.1% 非追蹤**（Case 42-A）；檔案裡不是孤例 —— Case 32（分享 2 ／ 儲存 1）打到 57.9%、Case 34 news-reaction（分享 6）48.9%。**這一 class 的天花板結構性落在 50-58%**，要上 83%（Case 37 分享 53）或 90%（Case 38 分享 195 ／ 儲存 466）一定要把公開分享與儲存兩個放大器點著。代價：話題退燒就沒了，**儲存只有 4** ＝ 沒有 revisit 理由，而且淨追蹤「資料不足」= 下游轉化近乎零（R7 真 KPI 貢獻 ≈ 0）。

**拉爆分享 4 招**：
1. 具體 demo + 看得到的成品 > 抽象宣稱（R36）
2. works-for-others > I'm great（第三方戰績，R38）
3. 免費 + 超有用 + 誠實三合一最 shareable（「我沒辦法教你直接開源丟你」）
4. R15 私訊分享設計「分享給寫 code 的朋友」觸發 essential 標籤

**Thread 對應信號 = 轉發**（見 R19，6/25 轉發 1,278 破紀錄（plateau））。

**評估用法**：分享 ≥ 20 = broke 鐵粉圈強訊號；mega 篇 ≥ 35。**配額**：無限（是 CTA / 評估指標不是公式）。

## R40：Dwell time（停留秒數）= 第三廣推信號 / 隱形王牌（提取自 formulas.md 訊號表）

FB 2026 三大跨粉絲圈信號 = 分享（R39）+ 留言（R16）+ **dwell time**。dwell 是「隱形王牌」因為它不顯示在面板但演算法重看。提取為獨立操作 / 評估規則。

**觸發時機**：設計內容結構 + 評估時的隱藏信號。

**🆕 面板可直接算的 dwell proxy：`瀏覽次數 ÷ 瀏覽者`**（= 平均每人看幾次，越高代表 revisit 越多）。實證帶：7/16 圖卡 news **2.06 次／人**（11,421 ÷ 5,552）> 7/16 CTA 圖卡 1.85 > 7/20 長文 1.84（33h）> Day 1 mega 約 1.7。圖卡型可能比長文型更會拉 revisit（⚠️ A 2.06 vs C 1.84 是 n=1 比較；B 1.85 與 C 1.84 差 0.014 屬雜訊，不構成第二證據）（讀完成本低、會回頭看留言），這解釋了 7/16 那篇互動只有 155 卻堆到 11,421 瀏覽。⚠️ 這是 proxy 不是官方 dwell，比值高也可能只是同一人被推播兩次，當趨勢看不要當硬指標。

**dwell time 組成**：
- 「查看更多」點擊（Day1 = 10,437 次）
- 圖 / 截圖放大（zoom）
- revisit（回來再看）

**拉高 dwell 的具體操作**：
1. **F12 時間戳直播**：「現在 22:47，3 小時後回來報結果」→ 未來式懸念觸發 revisit，單篇 revisit >30% 直接觸發「高價值內容」判定，比一次性回顧擴散強 2-3x
2. **F2 截圖先丟**：截圖 = 免費 dwell time，zoom/save 把你推到 Explore
3. **F15 完整原稿**：直接貼整套內容讓讀者滑很久（dwell 爆）
4. 前段留鉤子讓人點「查看更多」收合

**評估用法**：「查看更多」點擊數是 dwell proxy，高 = 演算法會加推。配 R18 儲存（revisit 信號）+ R6 4 指標看完整。

**配額 / 限制**：無（結構設計 + 評估規則）。狀態 emerging（dwell 不顯示面板，靠「查看更多點擊」+ 儲存間接量測）。

## R41：superlative hook 🧪 emerging（n=1，confound 未分離） —「最大 / 第一 / 史上最 / 破紀錄」= milestone / 成果 broke 放大器（6/14 實證）

從 F25b 段抽出獨立 hook 規則。同 milestone 集體歸功，加 superlative 權威宣稱 → 非追蹤 3.5x。

**鐵律**：發 milestone / 成果文時，能掛 superlative（「台灣最大」「全台第一」「史上最」「破紀錄」）就掛，**比純數字 broke 暴增**。「第一」會自己長腳，路人也想看「最大的長怎樣」。

**觸發時機**：
- 任何 milestone（社群人數 / 帳號成就 / 戰績）達到某個「可宣稱第一 / 最大」的位置
- 有 striking 具體數字（「100 個剪輯記憶」「史上最高儲存 466」）可當好奇 hook 時

**機制**：
1. superlative = 好奇 hook（「台灣最大的 AI Discord 長怎樣」）+ 權威信號雙觸發
2. 純數字（5000）對路人無 reference 框架 → 鐵粉才懂；「台灣最大」路人秒懂稀缺性
3. striking 具體細節（100 剪輯記憶）比抽象描述勾人 10 倍

**兩種 superlative（都是放大器）**：
- **外部 superlative**（vs 別人）：「台灣最大 / 全台第一」→ 6/14 broke 57.9%
- **內部 superlative**（vs 自己歷史）：「史上最高儲存 466 / 破 F19 紀錄 44 萬 / 轉發前高 15.7x」→ 破紀錄篇

**實證對比（同 F25b 集體歸功，唯一變數 = superlative）**：
| | 5/28 純數字（Line 5000）| 6/14 superlative（台灣最大）|
|---|---|---|
| 觀眾 | 852 | **3,079（3.6x）** |
| 非追蹤 | 16.7% 鎖鐵粉 | **57.9% broke** |

**具體操作**：
1. milestone 句先驗證 superlative **真實性**（真的是最大 / 第一？），真的才掛；掛在 hook 第一句最強
2. **內部 superlative 必綁具體當紅 artifact**才破天花板（抽象 stance 26 萬有天花板；綁「100 剪輯記憶」44 萬，見 R19 / Case 39）
3. superlative + 集體歸功（R32）+ milestone 數字三疊加最強

**配額 / 限制**：必須**真實**（誇大「第一」被打臉 = trust 崩）；同一 superlative 一個月別重複（「台灣最大」用過就換下個里程碑）；即使 Reel 格式弱（Case 32 排名 3/10）強 superlative text 仍帶得動 = 內容 > 格式。**純數字 milestone 鎖鐵粉是預設**，想 broke 三條救法：加 superlative（R41）/ 換第三方戰績（R38）/ 加集體 framing（R32）。

## R42：archetype 飽和天花板 —「強化 hook ≠ 換 archetype」（5/12 V3 fail 實證）

**強度：emerging（單 case 5/12 V3 實證，待第 2 個 archetype 飽和 case 升 proven）** — 與 R36-R41 proven 區隔。

**觸發時機**：同一 archetype（F6b 純血）月內已用 5-6+ 篇，或 viral 4 條件 AND 全綠但仍 plateau / 鎖鐵粉。

**機制**：連 6+ 篇 F6b 後鐵粉對「整體 voice」疲勞（不只 hype 詞）。F6b 公式本身**分享率天花板 = 0.21**，演算法 + 鐵粉雙重 saturation。**強化 hook（R11 金句 / R12 量化 / R13 反命題 / R14 punch 詞）是裝飾，不能突破 archetype 天花板**。

**實證**：5/12 V3 = viral 4 條件全綠 + R10 詞輪替 + R11-R14 全執行 → 仍 plateau 219 觀眾 / 11.3% 鎖鐵粉 / 0 留言 0 分享。同日 F19（換 archetype 到 F15 mini）→ 814 觀眾 / 29.8% broke。

**具體操作**：
1. F6b 月內 ≤ 2 篇（不同 intent，R5）且偵測到 voice 整體疲勞 → **換 archetype**：F15 mini（極簡開源）/ Mode C（F22 vulnerable / F23 行業洞察）/ 具體 demo（R36）
2. 別在 plateau 篇上加更多 hook 詞補救 ── 整篇重來換 archetype
3. archetype 輪播 > 單一 archetype 強化

**配額 / 限制**：是飽和偵測規則。**viral 4 條件全綠是必要不充分**，archetype 飽和 + Readability（meta 層數 ≤1）是兩個隱藏 fail 條件。

## 🎯 Viral 4 條件公式（核心，5 案例 AND 實證）

```
viral = 4 段 4 句結構 + 純血 voice + 全新敘事意圖（R5）+ 黃金時段（R4）
```

4 個 AND 不是 OR，任一缺 = 死。但 **Readability 是隱藏第 5 條件**（5/12 V3 四條全綠仍 fail，因 meta 4 層看不懂）。

## 📝 Readability 三項硬檢查（meta 層數上限統一 ≤ 1，解文件矛盾）

1. **5 秒非追蹤者讀懂測試**（3 句內懂）
2. **meta 層數 ≤ 1 = 最佳且為硬上限**（解先前 ≤1/≤2 並存矛盾，統一 ≤1）
   - meta 層數 = 讀者要疊幾層才懂這篇在講什麼；每多一層非追蹤者爬不過比例暴增
   - meta = 2 = 風險，只在鐵粉深化篇可接受（不指望 broke）
   - meta ≥ 3 = 直接重寫（Case 9 V3 四層 meta 全綠仍 219 fail；同日 F19 1 層 meta broke 814 / 29.8%）
3. **數字密度 ≤ 3 個/段**（超過 = dashboard report）

→ viral 4 條件全綠也不能跳過此檢查（隱藏第 5 條件）。
