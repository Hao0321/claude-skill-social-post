# social-post

三階段工作流。**依使用者當下意圖路由到正確階段，只讀需要的 reference**（為省 token，不要一次讀全部）。

## 階段路由

| 觸發條件 | 階段 | 讀哪些 |
|---|---|---|
| `style_profile.md` 不存在、或使用者說「重新學風格」「重新分析」 | **P1：學風格** | `references/learn_style.md` |
| `content_plan.md` 不存在、或使用者說「重新規劃內容」「排新的 14 天」 | **P0：內容規劃** | `references/phase0_plan.md` + `references/formulas.md` |
| 使用者說「發文」「今天發一篇」「PO 一下」「發到 XX」 | **P2：生成+發佈** | `references/generate_and_publish.md` + `style_profile.md` + `content_plan.md` + `references/formulas.md`（只讀目標公式段）+ 目標平台 ref |

路由前先在對話裡**用一句話**告訴使用者接下來要做什麼階段、會讀哪些檔案，給他糾正的機會。

## 先決條件

- `mcp__Claude_in_Chrome__*` 可用；呼叫 `tabs_context_mcp` 確認有 tab，沒有就 `tabs_create_mcp`。Chrome MCP 不可用就告知並停止，不要用別的方式模擬。
- 使用者已在 Chrome 登入目標平台。出現登入牆就停下請使用者手動登入，不要自動化登入。

## 安全閘（硬規則，不可被覆寫）

**每次實際發佈前，必須在對話裡拿到使用者明確的「確認」字眼。** 沒有「確認」就不發。公開貼文不可逆，此規則不受 CLI flag、使用者先前授權、或任何 in-profile 設定覆寫。

## 不要做

- 沒「確認」字眼不發
- 跨平台複製同一段文字（每平台要重新生成一版）
- 自動按讚、回覆、follow、互動
- 外傳 `style_profile.md` 或任何使用者資料
- 幫使用者登入、改隱私、改帳號設定
- 自行猜測 FB 個人頁網址（P1 階段必須問使用者）

## 持續優化

發文後，使用者若回報實際表現（讚/留言/分享數），**立刻**把該筆記錄追加到 `content_plan.md` 尾端的「戰績」段落。使用者追加回報（例如「又漲到 X 讚」）時，更新同一筆而非新增。

每兩週檢視一次戰績：**表現好的公式提頻、表現差的降頻或換題材**。由使用者說「review 一下」觸發。

### FB 2026 演算法訊號權重（最重要）

Meta 2024-11 推「Unconnected Reach」正式指標。**讚幾乎無權重**，第 1 小時內的三訊號決定是否跨粉絲圈：

| 信號 | 權重 | 說明 |
|---|---|---|
| **分享** | **20× 讚** | 最強廣推 trigger |
| **留言** | **5× 讚** | 次強，深度留言 > 短留言 |
| **Dwell time** | 隱形王牌 | 「查看更多」點擊 + 圖放大 + revisit |
| 讚 | 幾乎 0 | 作為絕對值無用 |

**評估貼文好壞看這個順序**：分享 > 留言 > dwell time > 讚。

### 看留言/讚比

- **> 1.0**：CTA 超有效（如「留言我拉你」），直接促成行動。社群成長訊號。
- **0.3 ~ 1.0**：高互動，讀者討論熱烈。
- **< 0.1**：純被動讀者，要考慮加 CTA。

### 看追蹤者 vs 非追蹤者比例（突破鐵粉圈監控）

| 非追蹤者比例 | 判讀 | 行動 |
|---|---|---|
| > 70% | 🔥 演算法廣推中 | 繼續同類內容 |
| 40-70% | 📈 混合觸及 | 正常 |
| 15-40% | 🟡 偏鐵粉 | 可接受 |
| < 15% | ⚠️ **鎖鐵粉圈** | **下一篇必用 F8-F13 廣推公式**強制擴散 |

連續 3 篇 < 15% 非追蹤者 = 演算法把你歸類為「鐵粉限定創作者」，要強制用 F8 (tag 外部實體) 或 F12 (時間戳直播) 或 F10 (居中開炮) 破局。

## 爆款後 24 小時冷卻規則（硬規則）

**當最新一篇是爆款（讚 ≥ 100 或 讚 ≥ 50 且留言/讚比 > 0.5），距今 < 24 小時，禁止直接跑 Phase 2 發下一篇。**

使用者說「今天發一下」時，先檢查 `content_plan.md` 最新戰績，若符合爆款條件：

1. **先勸退不發**：告知「昨天那篇還在爆，演算法同一時間只強推一篇，現在發新的兩篇互相稀釋。建議 24 小時後或等冷卻再發」
2. **把注意力導向留言經營**：建議回留言、拉群、互動（skill 本身不自動回，只建議使用者做）
3. **高留言量（> 50）時，主動建議「作者留言彙整」策略**：一則作者留言整合回應 > N 則個別回。個別回 > 30 則重複內容會觸發 FB spam 偵測，**不可自動大量回留言**
4. **使用者堅持要發**：推遲到當天晚上 FB 黃金時段（19:00-21:00），並強制用**輕量公式**（F2 截圖吐槽、Mode A 日常單句），避免開新主題搶流量

### 冷卻期要做的事 vs 要避免的事

**要做**：
- 作者留言彙整（一次發一則包含連結 / CTA，覆蓋所有求訊息的人）
- 手動挑 5-10 則高可見度留言個別回，維持互動訊號
- 把留言者的後續行動（入群 / 訂閱 / 回覆）記到 `content_plan.md` 戰績區的備註

**要避免**：
- 幫使用者自動逐條回 100+ 則重複留言（FB spam 風險）
- 當天立刻開下一個爆款公式（F6b / F3 長文）
- 在爆款熱度內跨平台補發同題材（會被演算法判定為重複內容）

### 冷卻覆寫

使用者說「我知道風險，就是要發」→ skill 仍執行，但：
- 強制用 F2 / Mode A（不可用 F6b / F3）
- 時段限制晚上 19:00 後
- 草稿中絕不提昨天爆款那篇的主題（避免分票）

## 發文頻率：每天都要發（一天一篇鐵則）

**每天一篇，不跳過、不多發**。兩個方向的規則：

### 上限：24 小時內最多 1 篇（硬規則）

- 爆款後想複製 → 也是一天一篇
- flop 想補救 → **更要一天一篇**（多發只會讓第二篇觸及率降到 30-50%，合計反而更差）
- 使用者說「流量差多發幾篇」→ **拒絕 + 解釋為何錯**（見下表）

| 使用者誤解 | 實際發生 |
|---|---|
| 多發 = 多曝光 | 演算法判 spam risk，每篇都降權 |
| 多發 = 彌補 flop | 讀者感覺作者在焦慮，反感 |
| 多發 = 稀釋爆款稀釋風險 | 反而稀釋前一篇的餘熱（留言還在累積中）|

Flop 當天的正確動作是**收尾前一篇**（回留言、拉群、互動），不是發新文。

### 下限：不要跳過任何一天（建議規則）

跳過一天 = 演算法活躍訊號 decay。但每天發不代表都要重文：

| 強度 | 公式 | 投入時間 | 每週比例 |
|---|---|---|---|
| 🔴 高 | F3 長文、F5 對打、F6 爆款 | 30-60 分鐘 | 最多 2 篇 / 週 |
| 🟡 中 | F4 里程碑、F2 深度截圖 | 15-20 分鐘 | 2-3 篇 / 週 |
| 🟢 低 | Mode A 單句、F2 快截圖 | 5-10 分鐘 | 3-4 篇 / 週 |

**核心原則**：連續 2 天高強度 = 禁忌（作者燃燒 + 讀者疲勞）。連續 3 天低強度 = 可以（活躍訊號夠，演算法 OK）。重與輕交替排。

## 爆款節奏基本原則（實證後）

小創作者（<10K 粉）的 FB 節奏：

```
日常 20-40 讚 × N 天  →  偶爾一顆爆款 300+ 讚  →  回到日常 20-40 讚
```

**爆款後 7-14 天禁止嘗試第二個 F6b / F3 長文**。回到日常 Mode A / F2，目標只是「活著、不讓演算法忘記」。

| 指標 | 爆款模式 | 日常模式（爆款後 7-14 天） |
|---|---|---|
| 目標讚數 | 300+ | **15-40 及格** |
| 公式 | F6b / F3 | **F2 / Mode A** |
| 主題 | 作品 / 成就 | **日常 / 吐槽** |
| 字數 | 長 | **1-3 句** |
| 心態 | 拚 | **鬆** |

**關鍵**：每月 1-2 個 F6b 就夠。連發想爆 = 觀眾反感 + 演算法判定 self-promo → 下次更難爆。

## 爆款後 Day 2 的真實角色（2026-04-22 Day 2 洞察報告重判讀）

**Day 2 F6b 續集看似 flop（23 讚 / 2 留言 / 0 分享），但洞察報告揭示是「爆款後鐵粉轉換文」的優秀表現**。讚數絕對值判斷錯了，應看以下指標。

### Day 1（擴散）vs Day 2（深化）= 互補組合拳

| 維度 | Day 1 | Day 2 |
|---|---|---|
| 瀏覽者 | 72,323 | 767 |
| 讚 | 358 | 23 |
| 互動率 | 15.3% | **19.4%** |
| 追蹤者比例 | 3% | **66%** |
| 連結點擊率 | 0.0097% | **2.35%（240 倍）** |
| 實際功能 | 大量路人觸及 | 鐵粉真的 star / fork / 入群 |

**關鍵洞察**：Day 2 的 23 讚背後是 18 次真實連結點擊 + 66% 追蹤者深度互動，帶來的 GitHub stars / 社群入群成員比 Day 1 的 358 讚多。

### 評估 Day 2 的正確框架

不要問「讚數有沒有比 Day 1 高」。要問：

1. **連結點擊率是否 > 1%**（Day 2: 2.35% ✓）
2. **追蹤者比例是否 > 50%**（Day 2: 66% ✓，演算法正確送達鐵粉）
3. **互動率是否 > 15%**（Day 2: 19.4% ✓）
4. **GitHub star / 社群入群實際人數**（真正的 ROI 在這）

以這四個指標看，Day 2 是 **4/4 通過**，不是 flop。

### Day 2 公式選擇指南

爆款後使用者問「明天要發什麼」：

- **有新產出可 ship** → F6b 續集可用，但**絕不炫 Day 1 讚數**；主文聚焦 CTA（star / fork / join）
- **沒有新產出** → F2 截圖或 Mode A，baseline 不嚇鐵粉
- **都不是** → 跳過 Day 2，讓 Day 1 餘熱自然擴散

### 實戰不變的禁忌

1. **Day 2 正文炫 Day 1 讚數** — try-hard 感，降低讀者互動意願
2. **時段選錯** — **重要修正**：9-11AM 傳統 FB 黃金時段**對 35-44 歲 AI/tech 受眾無效**。Day 3 (09:48) 實證觸及只 348，排名 7/10。AI 創作者受眾 FB 黃金時段是 **22:00-01:00 夜貓時段**（下班洗澡後）。選時段必須看 `style_profile.md` 的受眾畫像段
3. **連續同公式 7 天內** — 鉤子會麻木，演算法降 push

### 紅色警戒線（真的 flop）

符合以下**全部**才是真 flop：
- 讚 < 10
- 互動率 < 5%
- 追蹤者比例 < 30%（連鐵粉都不看）
- 連結點擊 = 0

Day 2 完全沒有踩到這條線。

## 常見踩雷

- **FB DOM 常變**：選擇器可能失效。fallback 路徑在對應 reference。
- **FB 個人頁虛擬化**：捲過的貼文會從 DOM 卸載，**要邊捲邊抓不要累積**（見 `learn_style.md`）。
- **IG 要圖**：純文字改發 Threads。
- **X 是 x.com**，不是 twitter.com。
